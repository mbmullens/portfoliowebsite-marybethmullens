Case Study: CHIRP


community radio player app

about CHIRP:

CHIRP (Chicago Independent Radio Project) is a volunteer-driven, listener supported community radio station formed in 2007 in Chicago, IL. They can be heard everyday on 107.1 FM in the city or worldwide online.

refreshing their app:

Originally created over ten years ago in 2013, the CHIRP app is due for a refresh, with an update to the more recent standards set out by Android and Apple's operating systems.

Did you know? When designing a native mobile app, it’s best to stick as closely as possible to each system's specified UI guidelines, which helps to ensure users of either mobile platform can intuitively start using the app upon open.

project brief:

Design two versions of a CHIRP native mobile app for Android (Material 3) & Apple (iOS 17):
1. Native mobile design patterns using each platform’s guidelines
2. Clearly defined typography & color
3. Consistent, intuitive UI design
4. User gestures demonstrated
5. Appropriate use of UI feedback
6. Designed for accessibility

key functions & user flow:

The most important function of the app is to play the latest song streaming live on CHIRP Radio. Based on research of other community radio apps. a few other key functions were defined but kept to a minimum:
1. Sign In or Create Account
2. Listen to What’s Playing Now
3. Request a Song
4. Save or Share Fav Songs
5. Read DJ Track Notes
6. Listen to Past Shows
7. Adjust Account/App Settings
8. View Events & Shop Merch
9. Donate to CHIRP

typography:

color contrast accessibility:

WCAG (Web Content Accessibility Guidelines) requires a minimum ratio of color contrast between fonts or icons and their background to accomodate people with visual disabilities. All colors in this app meet WCAG Level AA standards.

colors:

The same hues were used for both designs, starting with Apple's iOS recommended color palette then branching off into shades and tints of each color.

logo refresh:

The CHIRP bird logo was cleaned up and simplified for an app icon using Adobe Illustrator.
Their old album placeholder image was updated in Figma.

icon design:

Volume levels were designed as a silhouette of the Chicago skyline.
Icons and interfaces were designed in accordance with each platform’s guidelines.

mid-fidelity wireframes:

Grayscale prototypes developed in Figma.

high-fidelity wireframes:

Realistic prototypes were created in Figma, including interactive elements that would closely resembling the final app design for each platform.
