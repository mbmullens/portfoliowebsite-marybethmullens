Case Study: CampFam

camp booking website

UX/UI design of a responsive web app for booking family-friendly campgrounds.

view the interactive prototype

Quicklinks: Project Brief | Tasks | Competitors | Interviews | Personas | User Flow | Style Guide | Wireframes | Breakpoints


project brief:

Design a responsive web app for reserving public and private campsites in the U.S., aimed at families with kids.

tasks:

1. Research the User Experience of families looking for a place to camp.
2. Use that data to design the User Interface for all device sizes (starting with a mobile-first approach).
3. Prepare assets for hand-off to web-developers.

competitor research:

Analysis on two well-known camping reservation websites, ReserveAmerica.com and Hipcamp.com.

user interviews:

Interviews were conducted with adults age 25-55 who want to go camping with their children. Results showed everyone wants one easy website to search ALL campgrounds and find just the right one.

user personas:

Fictional profiles were created to keep user needs in mind.

key functions & user flow:

1. Be a one-stop-shop to find ALL public and private parks
2. Sort & filter search results
3. Offer detailed info, pics, video, maps, and Google reviews on each park
4. Let you book available sites
5. Post tips for taking kids camping

mood board:

Colors chosen evoke different elements of the outdoors, and the photography selected ties back into these hues as well as emphasize family bonding.

typography:

Museo Slab, a bold & clean type-writer font, was chosen for the header to convey a sense of nostalgia and a hint of child-like whimsy.

logo, icons & buttons:

Logos and icons were developed in Illustrator, and the rest of the interface was created in Figma.

low-fidelity wireframes:

Hand sketched with “Crazy-8” method (8 drawings in 8 minutes).

mid-fidelity wireframes:

Grayscale prototype developed in Figma.

high-fidelity wireframes:

Test out these wireframes in action with the Figma prototype!

view the interactive prototype

breakpoints:

Breakpoints are the building blocks of responsive design, used to control how the layout is adapted at different device screen sizes - XS, Small, Medium or Large.
